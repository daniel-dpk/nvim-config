# Remote Test Runner — Specification

## Purpose

Run Python project tests from Neovim (via neotest) **remotely** on a named
host while preserving neotest’s local UX:

* local discovery, remote execution
* fast iteration (stateful remote tree; only sync what changed)
* reproducible, host-selectable runs
* clean, minimal interface for neotest

---

## Command-line interface

Executable name: `remote_pytest` (Python recommended)

```
remote_pytest [--cwd <LOCAL_ROOT>] [--remote <NAME>] [--local]
              [--clean] [--dry-run] [--no-color]
              [--print-plan] [--print-config] [--verbose]
              -- <PYTEST-ARGV...>
```

### Flags

* `--cwd <LOCAL_ROOT>`
  Local project root (defaults to current working directory). Used to locate
  config and to map paths.

* `--remote <NAME>`
  Pick a remote from config. If omitted, use config `default`.

* `--local`
  Force a **local** run (ignore remote config).

* `--sync-only`
  Do everything except running the `pre_command` and actual test. This is
  usually the `setup_command` and `rsync`.

* `--clean`
  Before running, execute `clean_command` (if configured) on the remote, then
  proceed normally.

* `--dry-run`
  Resolve config/plan and print what would happen; **exit 0** without executing.

* `--half-dry`
  Similar to `--dry-run`, but communicate via `ssh` with the remote to query
  existence of directories. This allows deciding whether to run `setup_command`.
  Also, the `rsync` command is actually run, but with `--dry-run` argument to
  simulate the sync.

* `--no-color`
  Strip color codes from output (fallback for buggy terminals).

* `--print-plan`
  Print a one-line plan (host, remote_dir, python, args) before executing.

* `--print-config`
  Print the merged, effective config (after defaults + selected remote) for debugging.

* `--verbose`
  Log every command before execution; with `--dry-run` it only logs.

* `--`
  Required separator; everything after is passed verbatim as
  `<PYTEST-ARGV...>` (e.g., `python -m pytest -q tests/test_x.py::test_y`),
  typically generated by neotest-python.

**Exit code:** propagate the remote pytest exit code precisely.

**Signals:** on SIGINT/SIGTERM, forward to active rsync/ssh/pytest and exit
with the received signal code (so Neotest cancels cleanly).

## Configuration file

Filename at project root: **`.remote-tests.toml`**

### Top-level keys

```toml
# Which remote to use if --remote is not passed
default = "linux"

# Optional: force local runs (bypass remote entirely)
# force_local = true

[rsync]
# Exclusions when syncing local -> remote
exclude = [".git", ".venv", ".mypy_cache", ".pytest_cache", "__pycache__", "build", "dist"]
# Optional: include = ["pyproject.toml"]  # if explicit includes are needed
# Optional: extra = ["--delete-after", "--omit-dir-times"]  # extra raw rsync flags

[pytest]
args = ["-ra"]             # appended to argv supplied the caller
env  = { "PYTHONWARNINGS" = "default", "PYTHONUNBUFFERED" = "1" }  # exported on remote
color = "auto"             # "always" | "never" | "auto"

# Optional global path rewrite (rarely needed if we cd into remote_dir)
rewrite_prefix_remote = ""   # e.g., "/home/user/work/myproj"
rewrite_prefix_local  = ""   # e.g., "/Users/dan/src/myproj"

# Optional commands
setup_command = """\
# run only once when remote_dir is missing
git clone --depth=1 git@github.com:org/repo.git .
"""

clean_command = """\
# destructive cleanup invoked with --clean
rm -rf .venv build dist **/*.so **/*.pyd
"""

# Named remotes
[remotes.linux]
host = "mycluster"                 # SSH Host alias from ~/.ssh/config
# WARNING: Files may be deleted inside the remote_dir.
#          DO NOT use an existing directory with important data.
#          Run with --half-dry to check what rsync would change.
remote_dir = "~/mirror/myproj"     # directory to mirror into
python = "python"                  # interpreter (absolute path or name in venv)
pre_command = """\
# idempotent setup + activation
if [ ! -d .venv ]; then python -m venv .venv && . .venv/bin/activate && pip -q install -U pip setuptools wheel && pip -q install -e .; else . .venv/bin/activate; fi
python setup.py build_ext --inplace
"""

setup_command = """\
# run before pytest.setup_command when remote_dir is missing
"""

[remotes.altdistro]
host = "buildbox"
remote_dir = "~/src/myproj"
python = "~/.pyenv/versions/3.12.6/bin/python"
pre_command = "source ~/.bashrc && pyenv activate 3.12.6 && pip -q install -e . && python setup.py build_ext --inplace"
```

### Rules

* If **no config file** is found at/above `--cwd`: run **locally**.
* If `force_local = true`: run **locally**.
* `setup_command` runs **exactly once per remote**, when `remote_dir` does not exist.
* `clean_command` runs **only** when `--clean` is passed.
* `pre_command` runs **before every test execution** on the remote (typical
  place to activate venv, ensure editable install is in place, build Cython
  `--inplace`, etc.).
* The runner **must not** assume identical `$HOME` paths—use `remote_dir` and
  `--cwd` to map paths.

## Behavior (step-by-step)

1. **Resolve local root** (`LOCAL_ROOT`):

   * From `--cwd` or `pwd`.
   * Locate `.remote-tests.toml` by walking upward from `LOCAL_ROOT`.

     * If missing ⟶ **local run**.
   * Parse TOML (Python ≥3.11: `tomllib`; else `tomli` fallback).

2. **Local/remote decision**:

   * If `--local` or `force_local=true` ⟶ local run.
   * Else select remote: `--remote <NAME>` or `default`.

3. **Construct plan** (remote):

   * `HOST`, `REMOTE_ROOT`, `PYTHON`, `PRE_COMMAND`, `SETUP_COMMAND?`, `CLEAN_COMMAND?`
   * `PYTEST_ENV` (dict), `PYTEST_COLOR` policy
   * `RSYNC_EXCLUDES`, optional includes/extra flags
   * `PYTEST_ARGV` = argv after `--` (from neotest) **plus** `pytest.args`
     from config (append at end)

4. **print-plan / print-config** (if requested):

   * `--print-config`: dump effective config (after selecting remote).
   * `--print-plan`: log a single line, e.g.:
     `PLAN remote=linux host=mycluster root=~/work/myproj python=python args='-q -ra'`

5. **Initial setup**:

   * Check if `REMOTE_ROOT` exists:

     * If no: run `ssh HOST "mkdir -p REMOTE_ROOT"`; then run `setup_command`
       (if present) in that dir.

       * If `setup_command` absent, just proceed (rsync will create tree content).

6. **Optional clean**:

   * If `--clean`: run `clean_command` (if present) inside `REMOTE_ROOT`.

7. **Rsync sync (LOCAL_ROOT → HOST:REMOTE_ROOT)**:

   * Use `rsync -az --delete` by default (plus `rsync.extra` if any).
   * Apply `rsync.exclude` and optional `rsync.include`.
   * Use SSH multiplexing implicitly via user’s `~/.ssh/config`.
   * Keep remote repo fully stateful (no wipe) to preserve VCS metadata and
     build artifacts; rsync updates changed files only.

8. **Remote pre-run**:

   * Execute inside `REMOTE_ROOT`:
     `PRE_COMMAND` (if present). Common tasks:

     * create/activate venv if missing
     * install project in editable mode if first time
     * `python setup.py build_ext --inplace` (fast incremental)
     * any module loads (HPC `module load …`)
   * Export `pytest.env` vars. Respect `pytest.color`:

     * `auto`: use `ssh -tt` (pseudo-tty) so pytest auto-enables color.
     * `always`: set `PYTEST_ADDOPTS=--color=yes`.
     * `never`: set `PYTEST_ADDOPTS=--color=no`.

9. **Run pytest** (remote):

   * Command form:
     `ssh [-tt] HOST "cd REMOTE_ROOT && <export env> && <PRE_COMMAND> && <PYTHON> -m <PYTEST_ARGV…>"`

10. **Output handling**:

* Stream stdout/stderr line-buffered back to caller (preserve order).
* If both `rewrite_prefix_remote` and `rewrite_prefix_local` are non-empty,
  **rewrite absolute paths** in stdout/stderr accordingly (use safe string
  replace in Python, not regex; only replace full path prefixes).
* Honor `--no-color` by stripping ANSI codes (only if requested).

11. **Exit code & signals**:

* Return the **exact** exit code from the remote pytest process.
* On SIGINT/SIGTERM: propagate signal to active rsync/ssh, and exit with the signal code.

## Local fallback behavior

If running locally (no config, `force_local`, or `--local`):

* Execute `<PYTEST-ARGV…>` as given (do **not** append `pytest.args` from
  config since config may not exist).
* Preserve colors unless `--no-color`.
* Exit code: pytest’s exit code.

---

## Security & robustness

* **Never shell-eval local paths** inside remote commands. Always `cd` to
  `REMOTE_ROOT` then run configured strings.
* Treat `pre/setup/clean` as opaque shell snippets **on the remote**; do not
  interpolate local untrusted variables into them.
* Quote/escape `REMOTE_ROOT` and environment variable values safely when
  constructing remote command lines.
* Limit path rewriting to **prefix** replacement only (avoid accidental
  rewrites of non-path text).
* Provide clear errors for: missing remote name, unknown keys, missing
  `remotes.<NAME>`, SSH failures, rsync failures, and non-zero build commands.

## Performance notes

* Recommend users set SSH multiplexing in `~/.ssh/config`:

  ```
  Host mycluster
    ControlMaster auto
    ControlPath ~/.ssh/cm-%r@%h:%p
    ControlPersist 10m
  ```
* Use `rsync.extra` for `--delete-after` / `--omit-dir-times` if useful.
* Keep `pre_command` **idempotent** and fast on “no changes.”
* For large repos, consider `rsync --filter='merge <file>'` support later.

## Minimal Neotest glue (global)

* If `.remote-tests.toml` exists in (or above) the current project root, run
  with strategy `remote_cluster`; else run default (local).
* Optional: allow setting `vim.g.test_remote` to override `--remote <NAME>`.
